Apa itu Server Side Programming?

Server side programming adalah teknik pemrograman dimana logika aplikasi berada di server. Ketika pengguna melakukan permintaan ke server, server akan memproses permintaan tersebut, melakukan tugas yang diperlukan, dan mengirimkan hasilnya kembali ke klien.


Bahasa Pemrograman Server Side

Bahasa pemrograman server side adalah bahasa pemrograman yang dijalankan di sisi server untuk memproses logika aplikasi sebelum mengirimkan hasilnya ke client (biasanya browser). Berbeda dengan bahasa pemrograman client side yang dijalankan di browser, server side berjalan di komputer server.

Contoh Bahasa Pemrograman Server Side Populer

  • PHP – Bahasa scripting yang sudah lama dipakai untuk pengembangan web dinamis.
  • Python – Dengan framework seperti Django dan Flask, sangat populer untuk aplikasi web server side.
  • JavaScript (Node.js) – Ekstensi JavaScript agar bisa berjalan di server, sangat populer dan fleksibel.
  • Ruby – Dengan framework Ruby on Rails, dikenal untuk pengembangan web cepat.
  • Java – Sering digunakan untuk aplikasi enterprise dan aplikasi web besar menggunakan framework seperti Spring.

Bahasa-bahasa pemrograman diatas memiliki karakteristik dan kekuatan masing-masing, yang dapat digunakan sesuai kebutuhan aplikasi dan pengembang.

Kelebihan Bahasa Pemrograman Server Side

  • Mengelola dan menyimpan data secara aman di server.
  • Membuat halaman web dinamis yang disesuaikan dengan pengguna.
  • Mengelola autentikasi dan otorisasi pengguna.
  • Berinteraksi dengan database, memproses data dan bisnis logic.
  • Melindungi kode sumber dan rahasia logika dari pengguna.

Kekurangan Bahasa Pemrograman Server Side

  • Ketergantungan pada server.
  • Kurang fleksibel dibanding client server

Pemrograman Server Side

Pemrograman server side adalah proses pengembangan aplikasi web di mana logika dan pemrosesan data dilakukan di sisi server. Dalam arsitektur web, server bertanggung jawab untuk menangani permintaan dari klien (seperti browser) dan mengirimkan respons yang sesuai. Pemrograman server side memungkinkan pengembang untuk membuat aplikasi yang dinamis, interaktif, dan mampu mengelola data dengan efisien.

Pemrograman server side adalah proses pemrograman yang dijalankan di sisi server untuk menangani permintaan (request) dari client, memprosesnya, dan mengirimkan respons kembali. Proses ini melibatkan backend logic seperti pengolahan data, interaksi dengan database, dan rendering halaman web yang akan dikirimkan.

Cara Kerja Pemrograman Server Side

  1. Client (browser) mengirim permintaan HTTP ke server, misalnya untuk membuka halaman.
  2. Server menerima permintaan dan menjalankan kode program server side.
  3. Server memproses permintaan, misalnya membaca data dari database, menjalankan logika bisnis.
  4. Server membangun respons berupa halaman HTML, JSON, atau format lain.
  5. Server mengirimkan respons ke client.
  6. Client menerima respons dan menampilkan hasilnya.
Catatan: Komunikasi antara client dan server biasanya menggunakan protokol HTTP atau HTTPS.

Contoh Kode Program Server Side: Node.js dengan Express

Berikut contoh sederhana membuat server HTTP dengan Node.js dan framework Express yang melayani permintaan klien dan mengirimkan respons:

const express = require('express');
                    const app = express();
                    const PORT = 3000;

                    // Endpoint root
                    app.get('/', (req, res) => {
                    res.send('<h1>Halo dari server Node.js!</h1>');
                    });

                    // Jalankan server
                    app.listen(PORT, () => {
                    console.log(`Server berjalan di http://localhost:${PORT}`);
                    });

Penjelasan:

  • Kita buat server dengan memanggil express()
  • Mendefinisikan endpoint '/' untuk menerima GET request dan mengirimkan konten HTML sebagai respons.
  • Server berjalan dan mendengarkan di port 3000.

Anda bisa menjalankan kode ini dengan menyimpan di index.js, kemudian di terminal jalankan:

node index.js

Setelah server berjalan, buka http://localhost:3000 di browser untuk melihat hasilnya.


Web Server

Web server adalah sebuah software (perangkat lunak) yang memberikan layanan berupa data. Berfungsi untuk menerima permintaan HTTP atau HTTPS dari klien atau kita kenal dengan web browser (Chrome, Firefox). Selanjutnya ia akan mengirimkan respon atas permintaan tersebut kepada client dalam bentuk halaman web.

Fungsi Web Server

  • Mendengarkan permintaan dari client pada port tertentu (biasanya port 80 untuk HTTP atau port 443 untuk HTTPS).
  • Mengirimkan konten statis (file HTML, CSS, JavaScript, gambar) langsung ke client.
  • Menjalankan aplikasi server side (misalnya PHP, Python, Node.js) untuk memproses permintaan dinamis.
  • Mengelola koneksi dan protokol komunikasi antara client dan server.
  • Membersihkan berbagai cache yang terdapat pada penyimpanan serta semua dokumen yang tidak terpakai lagi.
  • Melakukan pemeriksaan terhadap sistem security yang berasal dari permintaan HTTP berdasarkan request klien atau web browser.
  • Menyediakan data berdasarkan request atau permintaan yang masuk agar dapat menjamin keamanan sistem yang berjalan dengan lancar.

Contoh Web Server Populer

  • Apache HTTP Server: Salah satu web server tertua dan paling banyak dipakai.

    • Web server yang populer dan paling banyak digunakan kebanyakan orang, yaitu jenis Apache. Pada awalnya Apache didesain guna mendukung penuh sistem operasi UNIX. Selain cukup mudah dalam implementasinya, Apache juga memiliki beberapa program pendukung sehingga memberinkan layanan yang lengkap, seperti PHP, SSI dan kontrol akses.

    • PHP (Personal Home Page atau PHP Hypertext Processor) : Program semacam CGI, berfungsi memproses teks yang bekerja di server. Apache sangat mendukung PHP dengan menempatkannya sebagai salah satu modulnya (mod_php). Hal tersebut membuat PHP bekerja lebih baik.
    • SSI (Server Side Include) : Perintah yang bisa disertakan dalam bekas HTML. Kemudian ia dapat diproses oleh web server ketika pengguna mengaksesnya.
    • Access Control : Kontrol Akses dapat dijalankan berdasarkan nama host atau nomor IP CGI (Common Gateway Interface). Lalu yang paling umum untuk digunakan adalah perl (Practical Extraction and Report Language), disupport oleh Apache dengan menempatkannya sebagai modul (mod_perl).
  • Nginx: Web server dan reverse proxy berperforma tinggi, populer sebagai load balancer.
  • Microsoft IIS: Web server dari Microsoft, biasanya digunakan dalam lingkungan Windows Server.

Cara Kerja Web Server dengan Server Side Code

Web server dapat melayani permintaan untuk file statis atau memproses kode server side.

  1. Client membuat permintaan HTTP ke web server.
  2. Jika permintaan untuk file statis (misalnya halaman HTML), web server mengirim file tersebut langsung.
  3. Jika permintaan untuk file dinamis (misalnya PHP, atau endpoint API), web server meneruskan permintaan tersebut ke interpreter/server side app.
  4. Server side program memproses permintaan dan memberikan hasil ke web server.
  5. Web server mengirimkan hasil ke client.

Contoh Konfigurasi Server Side dengan Apache dan PHP

Misalnya kita ingin melayani file PHP menggunakan Apache, Apache akan memanggil PHP interpreter untuk memproses file PHP dan mengirim hasil HTML ke browser.

Contoh kode PHP sederhana (file index.php):

<?php
                echo "<h1>Halo dari PHP Server Side!</h1>";
                ?>

Jika file ini berada di folder web server dan dikonfigurasi Apache dengan mod_php, maka saat diakses, server akan menjalankan PHP dan mengirim hasilnya ke client.

Ringkasan

  • Bahasa pemrograman server side adalah bahasa yang digunakan untuk menulis kode yang berjalan di server sebelum dikirim ke client.
  • Pemrograman server side mencakup pengolahan data, interaksi dengan database, dan logika aplikasi yang dijalankan di server.
  • Web server adalah perangkat lunak yang bertugas menerima permintaan client dan mengirimkan konten, baik statis maupun dinamis.
  • Contoh populer server side languages: PHP, Python, Node.js, Ruby, Java.
  • Contoh web server populer: Apache, Nginx, IIS.

Bahasa Pemrograman PHP

Kali ini kita akan belajar bahasa PHP sebagai pemrograman server. PHP (Hypertext Preprocessor) adalah bahasa pemrograman server side yang banyak digunakan untuk membangun aplikasi web dinamis. PHP berjalan di server dan menghasilkan halaman HTML yang dikirimkan ke client (browser).

PHP dirancang untuk kemudahan pembuatan halaman web dinamis, dan dapat disisipkan langsung ke dalam kode HTML. PHP sangat populer karena mudah dipelajari, bersifat open-source, dan memiliki banyak sumber daya serta komunitas.

1. Dasar PHP

Tag pembuka PHP adalah <?php dan tag penutup ?>. Semua kode PHP ditulis di antara tag ini.

<?php
                // Menampilkan teks
                echo "Halo Dunia!";
                ?>

2. Variabel dan Tipe Data

Variabel PHP diawali dengan tanda $ diikuti nama variabel. PHP mendukung tipe data seperti string, integer, float, boolean, array, dan objek.

<?php
                    $nama = "Andi";       // String
                    $umur = 25;           // Integer
                    $tinggi = 170.5;      // Float
                    $menikah = false;     // Boolean
                    ?>

3. Struktur Kontrol

3.1 Percabangan (if-else)

<?php
                    if ($umur >= 18) {
                        echo "Anda sudah dewasa.";
                    } else {
                        echo "Anda masih anak-anak.";
                    }
                    ?>

3.2 Switch

<?php
                    $warna = "merah";

                    switch ($warna) {
                        case "merah":
                            echo "Warna merah dipilih.";
                            break;
                        case "biru":
                            echo "Warna biru dipilih.";
                            break;
                        default:
                            echo "Warna tidak diketahui.";
                    }
                    ?>

3.3 Perulangan

While loop

<?php
                    $i = 1;
                    while ($i <= 5) {
                        echo "Angka: $i <br>";
                        $i++;
                    }
                    ?>

For loop

<?php
                    for ($i = 1; $i <= 5; $i++) {
                        echo "Angka: $i <br>";
                    }
                    ?>

4. Fungsi

Fungsi adalah blok kode yang dapat dipanggil berulang kali. Contoh membuat dan menggunakan fungsi di PHP:

<?php
                    function sapa($nama) {
                        return "Halo, " . $nama . "!";
                    }

                    echo sapa("Budi"); // Output: Halo, Budi!
                    ?>

5. Contoh Kode PHP Lengkap: Halaman Web Dinamis

Di bawah ini contoh file PHP yang menampilkan halaman HTML dengan penggunaan variabel dan kontrol:

<!DOCTYPE html>
                    <html lang="id">
                    <head>
                    <meta charset="UTF-8">
                    <meta name="viewport" content="width=device-width, initial-scale=1">
                    <title>Contoh Halaman PHP</title>
                    </head>
                    <body>

                    <?php
                    $nama = "Sari";
                    $umur = 22;

                    echo "<h1>Selamat datang, $nama!</h1>";

                    if ($umur >= 18) {
                        echo "<p>Anda sudah dewasa.</p>";
                    } else {
                        echo "<p>Anda masih anak-anak.</p>";
                    }
                    ?>

                    </body>
                    </html>

6. Cara Menjalankan File PHP

  • PHP harus dijalankan pada server yang mendukung PHP, seperti Apache, Nginx, atau dapat menggunakan server built-in PHP.
  • Untuk menjalankan local, Anda bisa instal paket seperti XAMPP, WAMP, LAMP, atau gunakan PHP built-in server dengan perintah: php -S localhost:8000 dari terminal di direktori file PHP Anda.
  • Buka browser dan akses URL sesuai alamat server (misal: http://localhost:8000/namafile.php).

PHP dan MySQL

PHP adalah bahasa pemrograman server side yang banyak digunakan untuk pengembangan aplikasi web dinamis. Sedangkan MySQL merupakan sistem manajemen basis data relasional (RDBMS) yang populer dan handal untuk menyimpan data secara terstruktur. Kombinasi PHP dan MySQL memungkinkan pengembang membuat aplikasi yang interaktif dan mampu mengelola data dengan efisien.

Koneksi PHP ke MySQL

Untuk menghubungkan PHP dengan MySQL, ada dua ekstensi utama yang dapat digunakan: mysqli dan PDO. Contoh berikut menggunakan mysqli untuk melakukan koneksi dan mengecek keberhasilan koneksi ke database:

<?php
                // Informasi koneksi database
                $host = "localhost";
                $user = "root";
                $password = "";
                $database = "contoh_db";

                // Membuat koneksi
                $conn = new mysqli($host, $user, $password, $database);

                // Cek koneksi
                if ($conn->connect_error) {
                    die("Koneksi gagal: " . $conn->connect_error);
                }
                echo "Koneksi berhasil!";
                ?>

Operasi CRUD dengan PHP dan MySQL

CRUD adalah singkatan dari Create, Read, Update, dan Delete yang merupakan operasi dasar dalam pengelolaan data di database. Berikut contoh penerapan operasi CRUD sederhana menggunakan PHP dan MySQLi:

Create (Tambah Data)

<?php
                // Menambah data ke tabel 'users'
                $name = "Budi";
                $email = "budi@example.com";

                $sql = "INSERT INTO users (name, email) VALUES ('$name', '$email')";
                if ($conn->query($sql) === TRUE) {
                    echo "Data berhasil ditambahkan";
                } else {
                    echo "Error: " . $conn->error;
                }
                ?>

Read (Ambil Data)

<?php
                // Mengambil data dari tabel 'users'
                $sql = "SELECT * FROM users";
                $result = $conn->query($sql);

                if ($result->num_rows > 0) {
                    while($row = $result->fetch_assoc()) {
                        echo "ID: " . $row["id"]. " - Nama: " . $row["name"]. " - Email: " . $row["email"]. "<br>";
                    }
                } else {
                    echo "0 hasil";
                }
                ?>

Update (Ubah Data)

<?php
                // Mengubah data dengan id tertentu
                $id = 1;
                $newEmail = "budi_baru@example.com";

                $sql = "UPDATE users SET email='$newEmail' WHERE id=$id";
                if ($conn->query($sql) === TRUE) {
                    echo "Data berhasil diupdate";
                } else {
                    echo "Error updating record: " . $conn->error;
                }
                ?>

Delete (Hapus Data)

<?php
                // Menghapus data dengan id tertentu
                $id = 1;

                $sql = "DELETE FROM users WHERE id=$id";
                if ($conn->query($sql) === TRUE) {
                    echo "Data berhasil dihapus";
                } else {
                    echo "Error deleting record: " . $conn->error;
                }
                ?>

Keamanan pada PHP dan MySQL

Salah satu perhatian penting dalam menggunakan PHP dan MySQL adalah keamanan, terutama untuk menghindari serangan SQL Injection. Cara yang direkomendasikan adalah menggunakan Prepared Statements yang memisahkan data dan perintah SQL sehingga data yang dimasukkan tidak bisa mengubah struktur query.

Contoh penggunaan prepared statement untuk menambahkan data:

<?php
                // Menggunakan prepared statement untuk insert data
                $stmt = $conn->prepare("INSERT INTO users (name, email) VALUES (?, ?)");
                $stmt->bind_param("ss", $name, $email);

                $name = "Andi";
                $email = "andi@example.com";
                $stmt->execute();

                echo "Data berhasil ditambahkan dengan prepared statement";

                $stmt->close();
                ?>

Tips

  • Selalu gunakan prepared statements untuk operasi database yang menerima input pengguna.
  • Rancang struktur database dengan indeks yang tepat untuk optimasi kinerja query.
  • Lakukan validasi dan sanitasi input pada sisi server untuk menjaga keamanan data.
  • Backup database secara berkala untuk menghindari kehilangan data.
  • Gunakan error handling yang tepat untuk memudahkan debugging dan meningkatkan stabilitas aplikasi.
