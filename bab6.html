Bab 6: Bahasa Pemrograman Server Side

Materi ini membahas secara lengkap mengenai bahasa pemrograman server side, konsep pemrograman server side, serta pemahaman tentang web server. Di dalamnya juga terdapat contoh kode sumber untuk memperkuat pemahaman Anda.

1. Memahami Bahasa Pemrograman Server Side

Bahasa pemrograman server side adalah bahasa pemrograman yang dijalankan di sisi server untuk memproses logika aplikasi sebelum mengirimkan hasilnya ke client (biasanya browser). Berbeda dengan bahasa pemrograman client side yang dijalankan di browser, server side berjalan di komputer server.

1.1 Contoh Bahasa Pemrograman Server Side Populer

  • PHP – Bahasa scripting yang sudah lama dipakai untuk pengembangan web dinamis.
  • Python – Dengan framework seperti Django dan Flask, sangat populer untuk aplikasi web server side.
  • JavaScript (Node.js) – Ekstensi JavaScript agar bisa berjalan di server, sangat populer dan fleksibel.
  • Ruby – Dengan framework Ruby on Rails, dikenal untuk pengembangan web cepat.
  • Java – Sering digunakan untuk aplikasi enterprise dan aplikasi web besar menggunakan framework seperti Spring.

Bahasa-bahasa ini memiliki karakteristik dan kekuatan masing-masing, yang dapat digunakan sesuai kebutuhan aplikasi dan pengembang.

1.2 Kelebihan Bahasa Pemrograman Server Side

  • Mengelola dan menyimpan data secara aman di server.
  • Membuat halaman web dinamis yang disesuaikan dengan pengguna.
  • Mengelola autentikasi dan otorisasi pengguna.
  • Berinteraksi dengan database, memproses data dan bisnis logic.
  • Melindungi kode sumber dan rahasia logika dari pengguna.

2. Memahami Pemrograman Server Side

Pemrograman server side adalah proses pemrograman yang dijalankan di sisi server untuk menangani permintaan (request) dari client, memprosesnya, dan mengirimkan respons kembali. Proses ini melibatkan backend logic seperti pengolahan data, interaksi dengan database, dan rendering halaman web yang akan dikirimkan.

2.1 Cara Kerja Pemrograman Server Side

  1. Client (browser) mengirim permintaan HTTP ke server, misalnya untuk membuka halaman.
  2. Server menerima permintaan dan menjalankan kode program server side.
  3. Server memproses permintaan, misalnya membaca data dari database, menjalankan logika bisnis.
  4. Server membangun respons berupa halaman HTML, JSON, atau format lain.
  5. Server mengirimkan respons ke client.
  6. Client menerima respons dan menampilkan hasilnya.
Catatan: Komunikasi antara client dan server biasanya menggunakan protokol HTTP atau HTTPS.

2.2 Contoh Kode Program Server Side: Node.js dengan Express

Berikut contoh sederhana membuat server HTTP dengan Node.js dan framework Express yang melayani permintaan klien dan mengirimkan respons:

const express = require('express');
const app = express();
const PORT = 3000;

// Endpoint root
app.get('/', (req, res) => {
  res.send('<h1>Halo dari server Node.js!</h1>');
});

// Jalankan server
app.listen(PORT, () => {
  console.log(`Server berjalan di http://localhost:${PORT}`);
});

Penjelasan:

  • Kita buat server dengan memanggil express()
  • Mendefinisikan endpoint '/' untuk menerima GET request dan mengirimkan konten HTML sebagai respons.
  • Server berjalan dan mendengarkan di port 3000.

Anda bisa menjalankan kode ini dengan menyimpan di index.js, kemudian di terminal jalankan:

node index.js

Setelah server berjalan, buka http://localhost:3000 di browser untuk melihat hasilnya.

3. Memahami Web Server

Web server adalah perangkat lunak yang berjalan di server untuk menerima permintaan (request) dari client melalui protokol HTTP atau HTTPS, kemudian melayani permintaan tersebut dengan mengirimkan halaman web, file statis, atau hasil pemrosesan program server side.

3.1 Fungsi Web Server

  • Mendengarkan permintaan dari client pada port tertentu (biasanya port 80 untuk HTTP atau port 443 untuk HTTPS).
  • Mengirimkan konten statis (file HTML, CSS, JavaScript, gambar) langsung ke client.
  • Menjalankan aplikasi server side (misalnya PHP, Python, Node.js) untuk memproses permintaan dinamis.
  • Mengelola koneksi dan protokol komunikasi antara client dan server.

3.2 Contoh Web Server Populer

  • Apache HTTP Server: Salah satu web server tertua dan paling banyak dipakai.
  • Nginx: Web server dan reverse proxy berperforma tinggi, populer sebagai load balancer.
  • Microsoft IIS: Web server dari Microsoft, biasanya digunakan dalam lingkungan Windows Server.

3.3 Cara Kerja Web Server dengan Server Side Code

Web server dapat melayani permintaan untuk file statis atau memproses kode server side.

  1. Client membuat permintaan HTTP ke web server.
  2. Jika permintaan untuk file statis (misalnya halaman HTML), web server mengirim file tersebut langsung.
  3. Jika permintaan untuk file dinamis (misalnya PHP, atau endpoint API), web server meneruskan permintaan tersebut ke interpreter/server side app.
  4. Server side program memproses permintaan dan memberikan hasil ke web server.
  5. Web server mengirimkan hasil ke client.

3.4 Contoh Konfigurasi Server Side dengan Apache dan PHP

Misalnya kita ingin melayani file PHP menggunakan Apache, Apache akan memanggil PHP interpreter untuk memproses file PHP dan mengirim hasil HTML ke browser.

Contoh kode PHP sederhana (file index.php):

<?php
echo "<h1>Halo dari PHP Server Side!</h1>";
?>

Jika file ini berada di folder web server dan dikonfigurasi Apache dengan mod_php, maka saat diakses, server akan menjalankan PHP dan mengirim hasilnya ke client.

Ringkasan

  • Bahasa pemrograman server side adalah bahasa yang digunakan untuk menulis kode yang berjalan di server sebelum dikirim ke client.
  • Pemrograman server side mencakup pengolahan data, interaksi dengan database, dan logika aplikasi yang dijalankan di server.
  • Web server adalah perangkat lunak yang bertugas menerima permintaan client dan mengirimkan konten, baik statis maupun dinamis.
  • Contoh populer server side languages: PHP, Python, Node.js, Ruby, Java.
  • Contoh web server populer: Apache, Nginx, IIS.
