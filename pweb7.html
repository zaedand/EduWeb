Framework

Materi

Definisi Framework Website

Framework adalah kerangka kerja. Framework juga dapat diartikan sebagai kumpulan script (terutama class dan function) yang dapat membantu developer/programmer dalam menangani berbagai masalah-masalah dalam pemrograman seperti koneksi ke database, pemanggilan variabel, file,dll sehingga developer lebih fokus dan lebih cepat membangun aplikasi. Bisa juga dikatakan Framework adalah komponen pemrorgaman yang siap re-use kapans aja, sehingga programmer tidak harus membuat skrip yang sama untuk tugas yang sama. Misalkan saat anda membuat aplikasi web berbasis ajax yang setiap kali harus melakukan XMLHttpRequest, maka Xajax telah mempurmudah untuk anda dengan menciptakan sebuah objek khusus yang siap digunakan untuk operasi Ajax berbasis PHP. Itu adalah salah satu contoh kecil, selebihnya Framework jauh lebih luas dari itu.

Secara sederhana bisa dijelaskan bahwa framework adalah kumpulan fungsi (libraries), maka seorang programmer tidak perlu lagi membuat fungsi-fungsi (biasanya disebut kumpulan library) dari awal, programmer tinggal memanggil kumpulan library atau fungsi yang sudah ada di dalam framerwork, tentunya cara menggunakan fungsi-fungsi itu sudah ditentukan oleh framework. Beberapa contoh fungsi-fungsi standar yang telah tersedia dalam suatu framework adalah fungsi paging, enkripsi, email, SEO, session, security, kalender, bahasa, manipulasi gambar, grafik, tabel bergaya zebra, validasi, upload, captcha, proteksi terhadap XSS(XSS filtering), template, kompresi, XML dan lain-lain.

Framework dapat mempercepat proses pengembangan, meningkatkan konsistensi kode, memastikan praktik terbaik diterapkan, dan memudahkan kolaborasi dalam tim pengembang.

Sejarah Singkat Framework Website

Sejarah framework website dimulai pada akhir 1990-an dan awal 2000-an, ketika pengembangan web mulai berkembang pesat. Pada masa itu, pengembang sering kali menulis kode dari awal tanpa struktur yang jelas, yang mengakibatkan kesulitan dalam pemeliharaan dan pengembangan aplikasi yang lebih besar.

Dengan meningkatnya kompleksitas aplikasi web, muncul kebutuhan untuk memiliki struktur yang lebih terorganisir. Framework pertama yang dikenal adalah Ruby on Rails, yang diluncurkan pada tahun 2005. Ruby on Rails memperkenalkan konsep Convention over Configuration dan Don't Repeat Yourself (DRY), yang membantu pengembang untuk menulis kode yang lebih bersih dan efisien.

Setelah keberhasilan Ruby on Rails, banyak framework lain muncul, seperti Django untuk Python, yang juga diluncurkan pada tahun 2005, dan ASP.NET untuk C#. Seiring waktu, framework JavaScript seperti AngularJS (dirilis pada tahun 2010) dan React (dirilis pada tahun 2013) menjadi populer, memungkinkan pengembang untuk membangun aplikasi web dinamis dan interaktif.

Saat ini, framework website telah menjadi bagian integral dari pengembangan web modern, dengan banyak pilihan yang tersedia untuk berbagai bahasa pemrograman dan kebutuhan aplikasi. Framework membantu pengembang untuk mempercepat proses pengembangan, meningkatkan keamanan, dan menerapkan praktik terbaik dalam pengembangan aplikasi.


CSS - Bootstrap

Bootstrap adalah framework CSS yang paling populer dan banyak digunakan dalam pengembangan front-end website. Dikembangkan oleh Twitter, Bootstrap menyediakan kumpulan komponen dan utilitas yang siap pakai untuk mempercepat proses pembuatan tampilan yang modern, responsif, dan konsisten di berbagai perangkat.

Bootstrap menawarkan sistem grid yang fleksibel, komponen UI seperti tombol, navigasi, formulir, modal, dan masih banyak lagi. Semua komponen ini dirancang untuk mudah digunakan dengan kelas-kelas CSS yang sudah ditentukan, sehingga developer tidak perlu menulis banyak kode CSS dari awal.

Contoh Struktur Folder Bootstrap (Front-end saja)

                bootstrap_project/
                │
                ├── css/                   # File CSS termasuk bootstrap.min.css
                ├── js/                    # JavaScript dan bootstrap.bundle.min.js
                ├── images/                # Gambar yang digunakan
                ├── index.html             # Halaman web utama
                └── assets/                # Aset tambahan (font, ikon)
Keunggulan Bootstrap
  • Mudah dipelajari dan digunakan.
  • Menghemat waktu dalam membangun UI yang responsif.
  • Dukungan yang luas dari komunitas dan dokumentasi lengkap.
  • Browser compatibility yang baik.
Penerapan Bootstrap
1. Tata Letak Responsif dengan Sistem Grid

Bootstrap menggunakan sistem grid 12 kolom yang memudahkan pengaturan layout yang fleksibel dan responsif. Contohnya:

<div class="container">
                <div class="row">
                    <div class="col-md-6">Kolom 1</div>
                    <div class="col-md-6">Kolom 2</div>
                </div>
                </div>

Pada layar sedang ke atas, akan ada dua kolom berukuran sama, sementara pada layar kecil, kolom akan menumpuk secara vertikal otomatis.

2. Membuat Navigasi dengan Navbar

Navbar Bootstrap responsif dan mudah dikostumisasi:

<nav class="navbar navbar-expand-lg navbar-dark bg-dark">
                <a class="navbar-brand" href="#">Logo</a>
                <button class="navbar-toggler" type="button" data-bs-toggle="collapse" data-bs-target="#navbarNav">
                    <span class="navbar-toggler-icon"></span>
                </button>
                <div class="collapse navbar-collapse" id="navbarNav">
                    <ul class="navbar-nav">
                    <li class="nav-item"><a class="nav-link active" href="#">Home</a></li>
                    <li class="nav-item"><a class="nav-link" href="#">About</a></li>
                    <li class="nav-item"><a class="nav-link" href="#">Contact</a></li>
                    </ul>

                </nav>
3. Membuat Tombol dan Komponen UI

Contoh tombol dengan berbagai varian:

<button type="button" class="btn btn-primary">Tombol Utama</button>
                <button type="button" class="btn btn-success">Tombol Sukses</button>
                <button type="button" class="btn btn-danger">Tombol Berbahaya</button>
4. Membuat Formulir yang Stylish

Bootstrap membuat styling form menjadi mudah dan konsisten:

<form>

5. Responsive Images

Bootstrap menyediakan kelas utility untuk membuat gambar responsif:

<img src="gambar.jpg" class="img-fluid" alt="Deskripsi gambar">
6. Menggunakan Modal (Popup)

Modal digunakan untuk menampilkan konten pop-up interaktif:

                    </div>
7. Responsive Utilities

Bootstrap menyediakan banyak kelas utilitas untuk kontrol margin, padding, visibilitas, dan positioning yang responsif. Misalnya:

<div class="d-none d-md-block">Hanya tampil di layar medium ke atas</div>

JavaScript - AngularJS

AngularJS adalah framework JavaScript open-source yang dikembangkan oleh Google untuk membangun aplikasi web dinamis dengan struktur yang terorganisir. AngularJS menggunakan arsitektur Model-View-Controller (MVC) yang memisahkan logika aplikasi, tampilan, dan data, sehingga memudahkan pengelolaan kode pada proyek web yang kompleks.

AngularJS dikembangkan oleh Misko Hevery dan dirilis pada tahun 2010. AngularJS dirancang untuk menyederhanakan proses pembuatan aplikasi web kompleks dengan menyediakan framework lengkap yang memungkinkan developer membangun aplikasi single-page (SPA) yang responsif. Pada tahun 2016, Google merilis Angular versi terbaru (disebut Angular 2+), yang merupakan perombakan penuh dari AngularJS.

Fitur Utama AngularJS
  • Data Binding Dua Arah (Two-Way Data Binding): Perubahan data di model langsung tercermin di tampilan (view) dan sebaliknya.
  • Dependency Injection: Memudahkan pengelolaan dependensi antar komponen.
  • Directive: Ekstensi HTML untuk membuat elemen baru, atribut, dan perilaku khusus.
  • Templates: HTML dengan markup tambahan yang di-binding dengan data model.
  • Routing: Membuat aplikasi single-page (SPA) yang dapat berpindah halaman tanpa reload.
  • Modularitas: Kode bisa diorganisasi menjadi modul-modul yang mudah dikelola.
Cara Memulai dengan AngularJS

Untuk menggunakan AngularJS, Anda dapat menyertakan pustaka via CDN dengan menambahkan script berikut di halaman HTML Anda:


                <script src="https://ajax.googleapis.com/ajax/libs/angularjs/1.8.2/angular.min.js"></script>
Contoh Struktur Folder Angular (JavaScript Frontend)

            angular_project/
            │
            ├── e2e/                   # End-to-end tests
            ├── node_modules/          # Package node yang diinstall
            ├── src/
            │   ├── app/               # Komponen, modul, service
            │   ├── assets/            # Gambar dan aset statis
            │   ├── environments/      # Pengaturan environment
            │   ├── index.html         # Halaman HTML utama
            │   └── main.ts            # Entry point aplikasi
            ├── angular.json           # Konfigurasi Angular CLI
            ├── package.json           # Dependensi dan script npm
            └── tsconfig.json          # Konfigurasi TypeScript
Contoh Sederhana AngularJS

Berikut contoh aplikasi kecil yang menampilkan pesan dinamis dengan AngularJS:


            <!DOCTYPE html>
            <html ng-app="myApp">
            <head>
                <title>AngularJS Example</title>
                <script src="https://ajax.googleapis.com/ajax/libs/angularjs/1.8.2/angular.min.js"></script>
            </head>
            <body>

            <div ng-controller="myController">
                <h5>{{ greeting }}</h5>
                <input type="text" ng-model="greeting">
            </div>

            <script>
                angular.module('myApp', [])
                .controller('myController', function($scope) {
                    $scope.greeting = "Halo, AngularJS!";
                });
            </script>

            </body>
            </html>
Penjelasan
  • ng-app mendefinisikan aplikasi AngularJS.
  • ng-controller mendefinisikan kontroler yang mengatur scope dalam elemen tersebut.
  • {{ greeting }} menampilkan data yang ada di scope secara dinamis (data binding).
  • ng-model mengikat input teks ke variabel greeting sehingga perubahan langsung tercermin di tampilan.

Panduan AngularJS

1. Setup Awal
a. Menyertakan AngularJS
<!-- Gunakan CDN -->
                <script src="https://ajax.googleapis.com/ajax/libs/angularjs/1.8.2/angular.min.js"></script>

                <!-- Atau file lokal -->
                <script src="js/angular.js"></script>
b. Struktur Dasar
<!DOCTYPE html>
                <html ng-app="myApp">
                <head>
                    <title>Aplikasi AngularJS</title>
                </head>
                <body>
                    <div ng-controller="MyController">
                        {{ pesan }}
                    </div>
                    
                    <script>
                        var app = angular.module('myApp', []);
                        app.controller('MyController', function($scope) {
                            $scope.pesan = "Halo AngularJS!";
                        });
                    </script>
                </body>
                </html>
2. Data Binding
<!-- Two-way binding -->
                <input type="text" ng-model="nama">
                <p>Halo, {{nama}}!</p>

                <!-- One-way binding -->
                <p ng-bind="nama"></p>
3. Directives Utama
<!-- ng-repeat untuk perulangan -->
                <ul>
                    <li ng-repeat="item in daftarItem">
                        {{item.nama}} - {{item.harga | currency}}
                    </li>
                </ul>

                <!-- ng-click untuk event -->
                <button ng-click="tambahItem()">Tambah</button>

                <!-- ng-show/ng-hide untuk tampilan -->
                <p ng-show="tampilkanPesan">Pesan tersembunyi</p>
4. Form Validation
<form name="myForm">
                <input type="email" name="email" ng-model="user.email" required>
                <span ng-show="myForm.email.$error.required">Email wajib diisi</span>
                <span ng-show="myForm.email.$error.email">Format email salah</span>
                
                <button ng-disabled="myForm.$invalid">Submit</button>
            </form>
5. HTTP Service
app.controller('DataController', function($scope, $http) {
                // GET request
                $http.get('/api/data')
                    .then(function(response) {
                        $scope.data = response.data;
                    });
                
                // POST request
                $scope.kirimData = function() {
                    $http.post('/api/save', $scope.formData)
                        .then(function(response) {
                            alert('Data tersimpan!');
                        });
                };
            });
6. Routing dengan ngRoute
<!-- Sertakan angular-route.js -->
                <script src="angular-route.min.js"></script>

                <!-- Tempat menampilkan view -->
                <div ng-view></div>
// Konfigurasi routing
                app.config(function($routeProvider) {
                    $routeProvider
                        .when('/', {
                            templateUrl: 'views/home.html',
                            controller: 'HomeController'
                        })
                        .when('/about', {
                            templateUrl: 'views/about.html',
                            controller: 'AboutController'
                        })
                        .otherwise('/');
                });
7. Custom Directive
app.directive('helloWorld', function() {
                    return {
                        restrict: 'E',
                        template: '<h1>Hello World!</h1>'
                    };
                });
<!-- Penggunaan -->
                <hello-world></hello-world>
8. Filter Kustom
app.filter('reverse', function() {
                    return function(text) {
                        return text.split('').reverse().join('');
                    };
                });
<!-- Penggunaan -->
                <p>{{ 'AngularJS' | reverse }}</p> <!-- Output: SJralugnA -->

PHP - Laravel

Laravel adalah framework PHP open-source yang dirancang untuk mempermudah pengembangan aplikasi web dengan sintaks yang elegan dan ekspresif. Laravel menyediakan berbagai fitur modern yang membantu pengembang membangun aplikasi yang scalable, aman, dan mudah dipelihara.

Laravel dibuat oleh Taylor Otwell dan pertama kali dirilis pada tahun 2011 sebagai alternatif modern bagi framework PHP yang sudah ada. Sejak itu, Laravel tumbuh pesat dan menjadi salah satu framework PHP paling populer di dunia, dengan komunitas besar dan ekosistem paket yang luas.

Fitur Utama Laravel
  • Routing yang Mudah: Laravel menawarkan cara yang sederhana untuk mendefinisikan rute aplikasi dan mengelola URL secara efisien.
  • ORM Eloquent: Object Relational Mapping (ORM) yang kuat memudahkan interaksi dengan database menggunakan model berbasis objek.
  • Migrasi Database: Memungkinkan pengelolaan skema database secara versi dan terstruktur.
  • Blade Templating: Sistem templating yang bersih dan mudah digunakan untuk membuat tampilan.
  • Middleware: Menyaring HTTP request masuk untuk keperluan autentikasi, logging, dan lainnya.
  • Sistem Otentikasi: Scaffold otentikasi lengkap untuk login, registrasi, dan manajemen pengguna.
  • Queue & Event: Mendukung pemrosesan tugas latar belakang dan event-driven programming.
Cara Memulai Proyek Laravel

Untuk membuat aplikasi Laravel baru, Anda harus menginstal Composer yang merupakan dependency manager PHP. Kemudian jalankan perintah berikut di terminal:

composer create-project --prefer-dist laravel/laravel nama_proyek

Perintah ini akan mengunduh Laravel dan membuat folder proyek baru dengan nama nama_proyek.

Membuat Route dan Controller Sederhana

Contoh mendefinisikan route di routes/web.php:

Route::get('/', function () {
                return view('welcome');
            });

Atau menggunakan Controller:

php artisan make:controller HomeController

Setelah membuat controller, tambahkan fungsi dan rute:

// routes/web.php
            Route::get('/home', [HomeController::class, 'index']);
            
            // app/Http/Controllers/HomeController.php
            public function index() {
                return view('home');
            }
Mengakses Database dengan Eloquent

Contoh Model User yang terhubung ke tabel users:

php artisan make:model User

Contoh menggunakan model untuk mengambil data:

// Di Controller
            $users = User::all();
            return view('user.index', compact('users'));
Blade Templating

Laravel menggunakan engine Blade untuk membuat template HTML dengan sintaks sederhana:

<!-- resources/views/user/index.blade.php -->
                <h1>Daftar Pengguna</h1>
                <ul>
                @foreach ($users as $user)
                <li>{{ $user->name }} ({{ $user->email }})</li>
                @endforeach
                </ul>
Artisan CLI

Laravel menyediakan Artisan, command line tool untuk otomatisasi seperti membuat controller, migrasi, dan menjalankan server lokal:

php artisan serve
                    php artisan make:controller NamaController
                    php artisan migrate

Panduan Laravel

Laravel sebagai framework PHP modern memiliki banyak fitur yang mempermudah pengembangan aplikasi web, mulai dari routing, pengelolaan basis data, blade templating, hingga sistem autentikasi. Berikut ini beberapa contoh penerapan Laravel yang umum digunakan dalam proyek.

1. Membuat Project Baru

Gunakan Composer untuk membuat project Laravel baru:

composer create-project --prefer-dist laravel/laravel nama_proyek
2. Menjalankan Server Development
cd nama_proyek
        php artisan serve

Server akan berjalan di http://localhost:8000.

3. Membuat Route dan Controller

Mendefinisikan route di routes/web.php:

Route::get('/halo', function () {
            return "Halo dari Laravel!";
        });

Atau membuat controller dengan artisan:

php artisan make:controller HaloController

Menambahkan method dan route:

// app/Http/Controllers/HaloController.php
        public function tampil() {
            return view('halo'); // mengarah ke resources/views/halo.blade.php
        }
        
        // routes/web.php
        Route::get('/halo', [HaloController::class, 'tampil']);
4. Membuat View dengan Blade

Membuat file resources/views/halo.blade.php:

<!DOCTYPE html>
        <html lang="en">
        <head>
            <meta charset="UTF-8">
            <title>Halaman Halo</title>
        </head>
        <body>
            <h1>Selamat datang di Laravel!</h1>
        </body>
        </html>
5. Pengelolaan Database dengan Migration

Membuat migrasi tabel baru:

php artisan make:migration create_posts_table

Mengedit file migrasi di database/migrations/ untuk membuat skema tabel

Schema::create('posts', function (Blueprint $table) {
            $table->id();
            $table->string('judul');
            $table->text('konten');
            $table->timestamps();
        });

Menjalankan migrasi:

php artisan migrate
6. Membuat Model dan Menggunakan Eloquent ORM
php artisan make:model Post

Contoh mengambil semua data posts di controller:


        use App\Models\Post;
        
        $posts = Post::all();
        return view('posts.index', compact('posts'));
7. Sistem Autentikasi

Laravel menyediakan paket Jetstream dan Breeze untuk implementasi autentikasi secara cepat dan aman. Berikut perintah untuk menginstall Breeze sebagai contoh:

composer require laravel/breeze --dev
        php artisan breeze:install
        npm install && npm run dev
        php artisan migrate

Setelah instalasi, Anda langsung mendapatkan fitur login, registrasi, reset password, dan lainnya.

8. Middleware dan Routing Group

Contoh penerapan middleware untuk proteksi route:

Route::middleware(['auth'])->group(function () {
            Route::get('/dashboard', function () {
                return view('dashboard');
            });
        });

Perbandingan Laravel dan CodeIgniter

Laravel dan CodeIgniter (CI) adalah dua framework PHP yang populer digunakan untuk pengembangan aplikasi web. Meskipun keduanya memiliki tujuan yang sama, yaitu mempermudah pengembangan aplikasi, mereka memiliki perbedaan signifikan dalam arsitektur, fitur, dan penggunaan. Berikut adalah perbandingan antara Laravel dan CodeIgniter.

1. Arsitektur
  • Laravel: Menggunakan arsitektur Model-View-Controller (MVC) yang memisahkan logika aplikasi, tampilan, dan data. Ini membuat kode lebih terstruktur dan mudah dipelihara.
  • CodeIgniter: Juga menggunakan pola MVC, tetapi lebih fleksibel dan tidak terlalu ketat dalam penerapan arsitektur. Ini memungkinkan pengembang untuk lebih bebas dalam mengatur struktur aplikasi.
2. Fitur
  • Laravel:
    • ORM Eloquent untuk interaksi database yang lebih mudah.
    • Routing yang lebih canggih dan dukungan RESTful.
    • Middleware untuk pengelolaan request dan autentikasi.
    • Blade templating engine yang kuat.
    • Fitur built-in untuk autentikasi, pengelolaan sesi, dan pengujian.
  • CodeIgniter:
    • Lebih ringan dan cepat, cocok untuk aplikasi kecil hingga menengah.
    • Pengaturan yang lebih sederhana dan mudah dipahami.
    • Fitur dasar untuk routing, database, dan session management.
    • Lebih sedikit fitur built-in dibandingkan Laravel, tetapi lebih mudah untuk di-custom.
3. Kemudahan Penggunaan
  • Laravel: Memiliki kurva belajar yang lebih curam karena banyaknya fitur dan konsep yang harus dipahami, seperti dependency injection, service providers, dan lain-lain.
  • CodeIgniter: Lebih mudah dipelajari bagi pemula, dengan dokumentasi yang jelas dan struktur yang sederhana.
4. Komunitas dan Dukungan
  • Laravel: Memiliki komunitas yang besar dan aktif, dengan banyak tutorial, forum, dan sumber daya online. Dokumentasi resmi sangat lengkap.
  • CodeIgniter: Juga memiliki komunitas yang baik, tetapi tidak sebesar Laravel. Dokumentasi cukup baik, tetapi tidak sekomprehensif Laravel.
5. Kinerja
  • Laravel: Meskipun lebih kaya fitur, Laravel bisa lebih berat dan memerlukan lebih banyak sumber daya server, terutama untuk aplikasi besar.
  • CodeIgniter: Lebih ringan dan cepat, sehingga lebih cocok untuk aplikasi yang memerlukan performa tinggi dengan sumber daya terbatas.
6. Penggunaan
  • Laravel: Cocok untuk aplikasi besar dan kompleks yang memerlukan banyak fitur dan skalabilitas.
  • CodeIgniter: Ideal untuk aplikasi kecil hingga menengah, prototyping, dan pengembangan yang cepat.

Pilihan antara Laravel dan CodeIgniter tergantung pada kebutuhan proyek dan preferensi pengembang. Jika Anda memerlukan framework yang kaya fitur dan siap untuk belajar, Laravel adalah pilihan yang tepat. Namun, jika Anda mencari sesuatu yang lebih sederhana dan cepat untuk aplikasi kecil, CodeIgniter bisa menjadi pilihan yang lebih baik.
